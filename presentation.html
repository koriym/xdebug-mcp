Debug Like a Pro: Conditional Tracing Power üéØ

Scenario: "Error only occurs when User ID is 0"

Traditional Way (Painful):

// Must pollute your code
if ($userId === 0) {
    var_dump($userData, $permissions, $sessionData);
    error_log("Debug: User ID 0 detected");
    // Risk of forgetting to remove in production...
}

Problems: Code pollution, production risks, manual cleanup

The xdebug-mcp Solution ‚ú®

Conditional Tracing (Smart):

# Wait silently until condition met, then start tracing
./vendor/bin/xdebug-debug --break='User.php:85:$userId==0' --exit-on-break -- php app.php

# Get complete execution history: how did we get $userId=0?
claude --continue "Analyze where userId=0 comes from and which process causes the issue"

Benefits: No code changes, complete context, production-safe

Another Example: "Crash Only When Array is Empty"

# Catch condition-specific problems with certainty
./vendor/bin/xdebug-debug --break='DataProcessor.php:42:empty($items)' --exit-on-break -- php batch.php

# AI analysis to identify root cause
claude --continue "Analyze the path to empty array and why it crashes with empty data"

Why This is Revolutionary:

  • ‚ùå Standard Xdebug: Breaks every time (inefficient)
  • ‚úÖ Conditional Tracing: Fast execution until problem, detailed recording only when needed
